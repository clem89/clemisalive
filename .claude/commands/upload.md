# 오늘 작업 동기화 (커밋 + 푸시 + 작업일지)

다음 순서대로 작업을 수행하세요. 각 단계를 순서대로 빠짐없이 진행하세요.

## 1단계: 변경사항 파악
`git status`와 `git diff HEAD`를 실행해서 오늘 변경된 파일과 내용을 파악하세요.

## 2단계: 작업일지 파일 생성
- 오늘 날짜를 `date +%Y-%m-%d` 로 확인
- `docs/작업일지/` 폴더가 없으면 생성
- `docs/작업일지/YYYY-MM-DD.md` 파일을 생성하고, 파악한 변경사항을 바탕으로 아래 형식에 맞게 오늘 작업 내용을 한국어로 작성

```markdown
# YYYY-MM-DD 작업일지

## 작업 내용
- (변경된 파일/기능 목록)

## 구현 상세
(각 기능에 대한 설명)

## 에디터 설정 필요 항목
(Unity 에디터에서 해야 할 작업이 있으면 기록, 없으면 생략)

## 다음 작업 후보
(이어서 할 만한 작업 아이디어)
```

## 3단계: 커밋
변경된 스크립트 파일과 작업일지 파일을 스테이징하고 커밋하세요.
- `git add Assets/Scripts/ docs/작업일지/ docs/기획/` 로 관련 파일만 스테이징
- 커밋 메시지는 한국어로, 오늘 작업의 핵심을 1줄로 요약
- 형식: `기능명: 세부내용 추가/수정/버그픽스`

## 4단계: 푸시
`git push` 로 원격 저장소에 업로드하세요.

## 5단계: 완료 보고
커밋 해시, 푸시 결과, 작업일지 경로를 사용자에게 알려주세요.
